盛水最多的容器

  • 语言： python3

  • 方法：双指针

解题思路

盛水容器，容量取决于桶的最短边以及底长，底长或最短边越长容量越大。

  • 暴力破解：算出每种情况底长下的所有容量，取最大值（时间复杂度太大

  • 遍历（一开始我个人的不成熟思路）：算出每个数值作为最短桶边下的最大容量，取最大值。没实现出来

  • 双指针（借鉴）：

    从最大底边开始，计算容量，之后移动最短边一个单位，计算容量，循环直至两边重合。

    • 移动最短边的原因：

      若移动最长边，底长减少但容器的高度不变（还是原来的最短边），容器容量只会减少，求不出最大值

    ps：妙啊

代码实现

取两个变量作为桶边位置，一个列表记录移动每一步后的容量：

left =0
right = len(height) -1
may_max =[]

循环主体：

while left != right :
            short = min(height[left],height[right])
            may_max.append(short * (right - left))
            if height[left] <= height[right]:
                left += 1
            else:
                right -= 1

循环条件为桶边位置不重合，在每次记录容量后用if..else移动桶边。

用append（）方法把每次移动后的容量记录在列表may_max[]里。

全部代码：

class Solution:
    def maxArea(self, height: List[int]) -> int:
        left =0
        right = len(height) -1
        may_max =[]

        while left != right :
            short = min(height[left],height[right])
            may_max.append(short * (right - left))
            if height[left] <= height[right]:
                left += 1
            else:
                right -= 1
        
        return max(may_max)

执行结果：

用时204ms，内存消耗：25.2mb

参照评论区某大佬的python代码，他没有用列表记录每一次的容量值，而是直接在每次循环中对比上次循环所得的容量值取最大的那一个。（减少内存消耗）

旋转图像

  • 语言：python
  • 方法：水平翻转+转置

代码实现

取矩阵的阶数

n = len(matrix)

水平翻转：

for i in range(n // 2):
            for j in range(n):
                matrix[i][j] , matrix[n - i -1][j] =  matrix[n - i -1][j] , matrix[i][j]

i遍历一半的行，j遍历所有的列，用x,y = y,x直接交换每一列上对应行的元素

转置：

for i in range(n):
            for j in range(i):
                matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]

转置操作只遍历矩阵下三角区域的元素，即遍历每个第m行的前m个元素，对这些元素与对应上三角区域的元素进行交换

全部代码：

class Solution:
    def rotate(self, matrix: list[list[int]]) -> None:
        n = len(matrix)

        for i in range(n // 2):
            for j in range(n):
                matrix[i][j] , matrix[n - i -1][j] =  matrix[n - i -1][j] , matrix[i][j]
        
        for i in range(n):
            for j in range(i):
                matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]

按序打印

  • 语言：python
  • 方法：上锁阻塞

解题思路

  1. 分析题目

    三个线程共用一个实例，调用同一个实例的三个方法，线程启用时顺序不定

    要求不管怎样的顺序调用三个方法，输出时永远是first()，second()，third()

  2. 方法思路

    用python中threading模块中的Lock类对线程上锁

    • 对Lock类中锁的理解：

      第一个获取Lock锁的线程可以继续执行程序，之后获取Lock锁的线程被阻塞

    由于是三个线程共用一个实例，所以可以在实例初始化的时候进行acquire()上锁但不解锁，在之后调用实例中的方法时用acquire()获取锁的时候就会被阻塞。

    利用了初始化和执行方法的前后关系在不知道线程名字、顺序的情况下让所有线程进行等待，只需在fist()函数中不acquire()获取锁就不会与实例初始化时的锁发生冲突，继而执行。

    之后依次解锁执行second()和third()方法

代码实现

import threading #导入threading模块

在Foo类实例化的时候创建锁并上锁：

def __init__(self):
        self.con1 = threading.Lock()#用于阻塞second()函数
        self.con2 = threading.Lock()#用于阻塞third()函数
        self.con1.acquire()
        self.con2.acquire()

对Foo类中first()函数的操作：

def first(self, printFirst: 'Callable[[], None]') -> None:
                        #没有再次获取锁，所以直接带着原来的锁继续执行
        printFirst()
        self.con1.release()#释放1锁，对second()阻塞取消

对Foo类中second()函数的操作：

def second(self, printSecond: 'Callable[[], None]') -> None:
        self.con1.acquire()#second()函数开始时获取1锁，但由于在实例化时已经获取过1锁，这里再获取时会被阻塞
        printSecond()      #first()函数结束后释放1锁，second()正常获取锁并执行    
        self.con2.release()#释放2锁，对third()阻塞取消

对Foo类中third()函数的操作：

def third(self, printThird: 'Callable[[], None]') -> None:
        self.con2.acquire()#third()函数开始时获取2锁，但由于在实例化时已经获取过2锁，这里再获取时会被阻塞
        printThird()      #second()函数结束后释放2锁，thrid()正常获取锁并执行
        self.con2.release()#释放2锁，良好习惯，防止死锁

全部代码：

import threading

class Foo:
    def __init__(self):
        self.con1 = threading.Lock()
        self.con2 = threading.Lock()
        self.con1.acquire()
        self.con2.acquire()

    def first(self, printFirst: 'Callable[[], None]') -> None:
        # printFirst() outputs "first". Do not change or remove this line.
        printFirst()
        self.con1.release()


    def second(self, printSecond: 'Callable[[], None]') -> None:
        self.con1.acquire()
        # printSecond() outputs "second". Do not change or remove this line.
        printSecond()
        self.con2.release()


    def third(self, printThird: 'Callable[[], None]') -> None:
        self.con2.acquire()
        # printThird() outputs "third". Do not change or remove this line.
        printThird()
        self.con2.release()

跳跃游戏

  • 语言：python
  • 方法：动态规划(迭代形式)
  • 参考视频：

解题思路

  1. 分析题目：

    看到题目是处理一个数组，感觉很类似在学习动态规划过程中看到的上面那个视频，在反复钻研那个视频后依次尝试了视频中的三个方法，只有最后一个方法合适并通过Leecode。下面来分析以下方法的优缺点

    1. 穷举法/暴力搜索
    2. 记忆化搜索/剪枝
    3. 改写成迭代形式

  2. 穷举法/暴力搜索

    没有通过Leecode原题，原因：超时

    所以直接上源代码：

    class Solution:
    def __init__(self):
        self.n =0
    def canJump(self, nums: list[int]) -> bool:
        step = nums[0]
        self.jump(nums,0,step)
        if self.n != 0:
            return True
        else:
            return False
    
    def jump(self,nums,indexes,step):
        if step >= len(nums)-1:
            self.n +=1
        for i in range(indexes+1 ,indexes + step+1):
            if i >= len(nums) -1:
                self.n += 1
                break
            else:    
                step =nums[i] 
                self.jump(nums,i,step)#递归

    大致思路是：

    1. 额外创建一个函数，该函数遍历数组某一元素能够到的所有位置，如果遍历到的位置是最后一个索引，记录下来，如果遍历不到最后一个位置，则对遍历的位置递归使用该函数，直到递归到最后一个元素。
    2. 最后主函数检测记录里有咩有遍历到最后一个元素，有的话返回True，没有返回False
    • 优点：可以记录从第一个索引位置出发有多少条路到达最后一个下标
    • 缺点：多次使用递归以及遍历以及重复计算同一个位置是否能到最后一个下标，耗时长

  3. 记忆化搜索/剪枝

    没有通过Leecode原题，原因：超时；超时的原因：超出递归深度（998）

    class Solution:
    def __init__(self):
        self.n =0
        self.memo = {}
    def canJump(self, nums: list[int]) -> bool:
        step = nums[0]
        self.jump(nums,0,step)
        if self.n != 0:
            return True
        else:
            return False
    
    def jump(self,nums,indexes,step):
        if indexes in self.memo:
            return 0
        if step >= len(nums)-1:
            self.n +=1
        for i in range(indexes+1 ,indexes + step+1):
            if i >= len(nums) -1:
                self.n += 1
                break
            else:    
                step =nums[i] 
                self.jump(nums,i,step)
        self.memo[indexes] = 1

    方法和上一个暴力搜索差不多，就是用了一个数组memo记录遍历数组nums时对元素是否能到最后一个下标的判断。

    • 优点：继承上个方法的优点，但减少了重复计算
    • 缺点：继承上个方法的缺点，遇到一万长度的数组（测试用例77），则会超出递归深度（最大998）。

  4. 迭代形式

    之前方法中的函数要使用递归是因为要判断后面的元素，那么用迭代直接先判断后面的元素就可以避免递归。

    这种思路也可以理解为去判断从最后一个下标开始，能否到达第一个下标。

    class Solution:    
    def canJump(self, nums: list[int]) -> bool:
        #传入数组的长度
        n = len(nums) 
        #记录nums数组可以到达末尾下标的位置索引
        p = 0 #L数组中的小指针
        L =[0]    #能到达最后一个下标的位置
        #使用reversed()方法对nums数组逆顺序遍历
        for i in reversed(range(n)):
            if nums[i] + i >= L[p]:#判断一个位置能否够到 能到达最后一个下标的 最靠近该位置的 索引
                L.append(i)    #记录该位置
                p += 1    #小指针右移
        #判断能到达最后一个下标的位置里有没有第一个下标
        if L[p] == 0:
            return True
        else:
            return False
